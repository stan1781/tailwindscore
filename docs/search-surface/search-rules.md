# Search Rules

## Goal

TailwindScore search should feel like a premium commerce discovery surface.

It is a calm product-finding layer, not an enterprise search platform.

## Supported Structure

- header-integrated search triggers
- global search overlay
- mobile search drawer behavior
- lightweight predictive discovery
- recent and popular search support

## Hierarchy

1. Search starts as a subtle shell action.
2. The surface should privilege discovery over configuration.
3. Product results lead; category and suggestion cues support.
4. Empty state should still offer a composed browsing path.

## Unsupported Features

- Algolia-style enterprise UI
- AI search
- dense dashboard filtering
- full SPA search apps
- oversized discovery overlays
