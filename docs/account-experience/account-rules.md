# Account Rules

- Build the customer account as a post-purchase surface, not a dashboard product.
- Keep one clear hierarchy: account header, customer navigation, primary content, restrained utility.
- Prefer cards, stacked summaries, and form sections over oversized data tables.
- Preserve WooCommerce compatibility through SSR templates and native endpoints.
- Keep overview content light; no analytics panels, backend widgets, or CRM framing.
- Use the same motion, spacing, and feedback language already established in checkout and cart.
