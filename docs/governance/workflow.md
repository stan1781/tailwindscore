# Documentation Governance Workflow

## Workflow Order

1. Read `docs/AI-ENTRY.md`.
2. Confirm the current phase and active boundaries.
3. Check whether the requested change belongs to constitution, workflow, reference, or archive.
4. Consolidate existing material before creating a new markdown file.
5. Archive superseded material in the same pass when a new canonical doc replaces it.
6. Update health metrics and the top-level map when the boundary changes.

## Canonical Layers

- Constitution: stable rules and boundaries
- Workflow: how work moves
- Reference: lookup material
- Archive: historical records

## Default AI Reading Rule

AI should read the phase files, AI constitution, governance workflow, and runtime boundaries first.

AI should not browse topical or historical documentation until the active files point there by name.

## Closure Rule

A documentation task is not closed until:

- the canonical destination is clear
- duplicated narrative is reduced
- any superseded document is archived or frozen
- the docs map and health metrics reflect the new state
