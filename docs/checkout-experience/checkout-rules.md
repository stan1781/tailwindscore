# Checkout Rules

- Checkout is a single calm purchase surface, not a dashboard or app shell.
- SSR owns hierarchy, summary, payment structure, and empty states.
- JavaScript only enhances loading rhythm, payment reveal, summary refresh, and focus recovery.
- Validation language stays restrained, field-specific, and consistent with the feedback system.
- Unsupported patterns: enterprise density, gamified progress UI, SPA checkout architecture, oversized payment cards.
