# Toast Rhythm

## Role

Toasts are reserved for lightweight confirmation after a completed commerce action.

## Timing

- Default dwell: about 4.2 seconds.
- Catalog and PDP success can dismiss automatically.
- Reduced motion keeps the same information with minimal transition.
- Error toasts should be rare and should usually pair with an inline state when recovery is needed.

## Copy

- Good: `Added to bag`
- Good: `Bag updated`
- Good: `Removed from bag`
- Avoid: emotional slogans, exclamation-heavy language, loyalty-program tone, gamified reward language

## Layout

- One compact surface per event
- No icon parade
- No progress bars
- No corner pileup beyond what the viewport can read comfortably
