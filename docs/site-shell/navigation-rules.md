# Navigation Rules

## Goal

The TailwindScore site shell navigation should feel calm, premium, and editorial.

It is a commerce navigation foundation, not an application router and not a navigation builder.

## Supported Structure

- desktop primary navigation
- mobile drawer navigation
- nested navigation up to 3 levels
- utility actions for search, account, and cart
- keyboard support for submenu triggers and sibling traversal

## Hierarchy Rules

1. Keep top-level navigation short and brand-first.
2. Use nested items for curation and category refinement, not for giant taxonomies.
3. Avoid equal visual weight across every item.
4. Utility actions stay separate from the editorial browsing layer.

## Interaction Posture

- Desktop navigation uses restrained hover and focus states.
- Submenus should feel like lightweight editorial reveals.
- Mobile navigation uses a drawer, but it should still feel like a website shell.
- Do not mimic app tabs, app bars, or marketplace toolbars.

## Unsupported Behaviors

- mega menu builders
- full-screen app navigation patterns
- SPA route handling
- router-like active-state orchestration
- oversized promo-heavy utility bars
- dense marketplace headers
