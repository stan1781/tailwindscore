# Icon Rules

## Goal

TailwindScore icons should feel calm, precise, and premium.

The system is SVG-first, SSR-first, and intentionally small.

## Style Direction

- Heroicons outline posture
- restrained Apple-like precision
- premium commerce utility
- monochrome by default

## Rules

1. Use one icon language across commerce, shell, and editorial UI.
2. Default stroke weight stays restrained and consistent at `1.5`.
3. Icons should feel optically balanced at small sizes.
4. Hover feedback should come from the container more than the icon itself.
5. Icons should support accessibility without depending on motion or color.

## Naming

- use semantic names
- prefer `chevron-down` over ambiguous aliases
- reserve social names for brand glyphs only

## Unsupported Styles

- icon fonts
- Font Awesome
- mixed outline and filled families
- colorful app-style iconography
- heavy hover transforms
- oversized toolbar icon buttons
- thick-stroke icon styles
