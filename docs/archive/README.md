# Archive Boundary

`docs/archive/` stores historical material that must not participate in default implementation context.

## Buckets

- `specs/`: completed and superseded design records
- `plans/`: completed and superseded implementation plans
- `audits/`: closed or historical audit records
- `legacy/`: old structure or old phase material retained for reference

## Read Rule

Archive is read by exception only.

Archive docs cannot hold canonical authority.

Use archive only when:

- the active phase explicitly points to it
- a documentation blocker requires historical evidence
- the user asks for historical context
